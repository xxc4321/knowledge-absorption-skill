# Knowledge Absorption - 知识吸收优化器

> 让AI用你懂的方式，帮你快速理解陌生领域并做出决策。

## 为什么需要这个工具？

你是否有过这样的经历：

- 想了解一个新技术（比如RAG、向量数据库），但看完教程还是云里雾里
- 只是想选一个适合自己的工具，却被一堆术语淹没
- 问AI问题，得到的回答像教科书，每个字都认识，连起来就是不懂

**问题不在你，而在讲解方式。**

大多数教程是为"学习级吸收"设计的（几十小时掌握原理），但你真正需要的可能是"决策级吸收"（15分钟做出选择）。

---

## 核心理念

| 目标 | 学习深度 | 所需时间 | 核心产出 | 适用场景 |
|-----|---------|---------|---------|---------|
| 学习级吸收 | 能解释原理、动手实现 | 几十到几百小时 | 技能 | 成为领域从业者 |
| 决策级吸收 | 能区分优劣、做出选择 | 15-60 分钟 | 判断 | 使用工具或服务 |

**本Skill专为"决策级吸收"设计。**

---

## 功能特性

### 🎯 双流程设计

**决策级吸收流程（15分钟法）**
- 问题定义与边界切割 → 砍掉90%无用信息
- 框架扫描与分类提取 → 只看分类，不看细节
- 类比翻译与概念降维 → 用你熟悉的领域做类比
- 决策维度提取与权重分配 → 你说了算
- 决策矩阵构建与结论得出 → 有据可依

**解释重构流程（4层结构）**
- 概念层级检测 → 先解释子概念
- 四层输出 → 类比→本质→术语→场景
- 快速验证 → 用户用自己的话说一遍
- 多轮优化 → 最多3轮，失败就人工介入

### 🧠 个性化适配

- 读取用户认知画像（思维偏好、学习背景、熟悉领域）
- 优先使用用户熟悉的领域做类比
- 避免使用用户不擅长的领域（如历史地理）

### 📈 跨会话知识积累

- 成功类比库：记录成功的类比，下次优先使用
- 类比失败记录：记录失败的类比，下次避免
- 用户场景库：快速匹配推荐方案

## 使用方法

### 触发方式

**自动触发**：
- 说"听不懂"、"不明白" → 触发解释重构
- 说"XX和XX哪个好"、"怎么选择" → 触发决策级流程

**手动触发**：
- 加载 `knowledge-absorption` 技能后开始对话

### 首次使用

首次使用会引导完成认知画像初始化：

1. **思维偏好**：文科/理科/混合型
2. **学习背景**：擅长/不擅长的学科
3. **当前熟悉领域**：你最懂什么
4. **解释偏好**：案例/逻辑/可视化/动手操作
5. **最近"听不懂"的例子**：用于校准

### 示例对话

```
用户：我想搭建个人知识库，不知道选哪种方式

AI：我来帮你选。先确认几个问题：
- 使用场景：个人/团队？
- 技术水平：小白/有基础/技术人员？
- 最看重的：省时间/省成本/数据安全/功能强大？

用户：个人用，小白，想省时间，数据也要安全

AI：个人知识库的主流方案：

1. 传统笔记软件（Notion、Obsidian）- 基于文件夹和标签
2. 本地RAG方案（AnythingLLM、Cherry Studio）- 本地部署，数据安全
3. 云端RAG服务（Notion AI、ChatGPT Custom GPT）- 开箱即用
4. 个人Wiki（Wiki.js、BookStack）- 结构化知识体系

你可能会遇到几个概念，我解释一下：

RAG = 聪明店员+库存系统
就像你问店员"店里有几款Dimoo"，他会先查库存系统再回答你，而不是瞎编。

...（继续输出决策矩阵和推荐结论）
```

## 核心方法论

### 为什么听不懂？

| 原因 | 占比 | 修复方案 |
|-----|------|---------|
| 缺少"问题上下文" | 45% | 先说明"解决了什么痛点"，再解释概念 |
| 专有名词"知识压缩" | 30% | 每个术语单独解释，或用类比替代 |
| 认知过载 | 20% | 一次只讲1-2个新概念 |
| 知识储备不足 | 5% | 从用户熟悉领域切入 |

### 类比模板示例

**AI类**：
- LLM → 过目不忘的店员（只懂培训时学过的）
- RAG → 聪明店员+实时库存系统
- Embedding → 给文字拍"语义照片"
- Agent → 会自己干活的店员

**数据存储类**：
- 数据库 → 仓库
- 向量数据库 → 图书馆的智能索引系统
- 缓存 → 前台货架
